# 前后端接口字段对齐

## 对比分析

### 1. `ChartConfig` 类型 — 最大不匹配

**前端定义** ([frontend/src/types/index.ts](frontend/src/types/index.ts)):
```typescript
interface ChartConfig {
  chart_type: "bar" | "line" | "pie" | "scatter" | "table";
  title: string;
  option: Record<string, unknown>;
}
```

**后端实际返回** (SSE `chart` 事件 / `MessageOut.chart_config`):
后端输出的是 **原生 ECharts option JSON**，结构是：
```json
{
  "title": {"text": "各商品类别总销售额"},
  "xAxis": {"type": "category", "data": [...]},
  "yAxis": {"type": "value"},
  "series": [{"type": "bar", "data": [...]}]
}
```
即直接是一个 ECharts option 对象，**没有** `chart_type`、`title`(顶级字符串)、`option` 这三层包装。

**影响范围**:
- `ChartConfig` 类型定义需要重构
- `ChartRenderer` 组件：目前用 `config.option` 传给 ECharts，改为直接传 `config`
- `Visualization` 组件：用 `chart.chart_type` 做图标映射、`chart.title` 做标签，需要从 ECharts option 中提取
- `chatStore.setLastAssistantChart`：参数类型跟着改
- `chartStore`：存储的类型跟着改
- `mockData.ts`：mock 数据结构需要调整

### 2. `SSEEvent` 类型 — 缺少 `data` 事件类型

**前端定义**:
```typescript
interface SSEEvent {
  type: "text" | "sql" | "chart" | "error" | "done";
  data: string;
}
```

**后端实际**: SSE 事件有 6 种类型 `text | sql | data | chart | error | done`，前端缺少 `data` 类型（查询结果原始数据）。

### 3. `Session` / `SessionOut` — 匹配

| 字段 | 前端 | 后端 | 状态 |
|------|------|------|------|
| id | string | str | OK |
| title | string | str | OK |
| created_at | string | str | OK |
| updated_at | string | str | OK |

### 4. `Message` / `MessageOut` — `chart_config` 类型不匹配

| 字段 | 前端 | 后端 | 状态 |
|------|------|------|------|
| id | string | str | OK |
| session_id | string | str | OK |
| role | "user"/"assistant" | str | OK |
| content | string | str | OK |
| sql | string? | str? | OK |
| chart_config | ChartConfig? | dict? | **不匹配**（前端包了一层 `{chart_type, title, option}`，后端直接是 ECharts option） |
| created_at | string | str | OK |

### 5. `ChatRequest` — 匹配

前端发送 `{ session_id, message }` 与后端 `ChatRequest` 一致。

## 修改清单（以后端为准）

### 文件 1: [frontend/src/types/index.ts](frontend/src/types/index.ts)

- **`ChartConfig`**: 改为直接表示 ECharts option 对象（`Record<string, any>`），或定义为带可选 title/series 的宽松类型
- **`SSEEvent.type`**: 增加 `"data"` 类型

- 所有 `chart_config` 从三层结构 `{chart_type, title, option: {...}}` 改为直接的 ECharts option `{title: {text: "..."}, xAxis: {...}, series: [...]}`

### 文件 3: [frontend/src/store/chatStore.ts](frontend/src/store/chatStore.ts)

- `setLastAssistantChart` 的参数类型从 `ChartConfig` 改为新类型

### 文件 4: [frontend/src/store/chartStore.ts](frontend/src/store/chartStore.ts)

- `charts` 数组元素类型从 `ChartConfig` 改为新类型

### 文件 5: [frontend/src/components/Visualization/ChartRenderer.tsx](frontend/src/components/Visualization/ChartRenderer.tsx)

- 从 `config.option` 改为直接传 `config` 给 `ReactECharts`

- Tab 标签的图标：从 `chart.chart_type` 改为从 `chart.series[0].type` 推断图表类型
- Tab 标签的标题：从 `chart.title` (string) 改为从 `chart.title.text` 提取

### 文件 7: [frontend/src/components/ChatArea/index.tsx](frontend/src/components/ChatArea/index.tsx)

- `handleSend` 中 mock 的 `addChart(...)` 调用需要适配新结构（Phase 4 会整体替换，这里先对齐类型）
